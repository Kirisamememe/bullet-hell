# 弾幕煉獄 (Danma Rengo) - 設計仕様書

## 概要

ブラウザで動作する縦スクロール弾幕STG。Bun + TypeScript + HTML5 Canvas で構築。東方Project に影響を受けた高密度な弾幕と、レトロピクセルアートのビジュアルスタイル。PC（キーボード）とスマートフォン（タッチ）の両方に対応。

## 技術スタック

- **ランタイム**: Bun (latest)
- **言語**: TypeScript (strict mode)
- **描画**: HTML5 Canvas（依存ゼロ、自前ゲームエンジン）
- **バンドル**: Bun のネイティブビルド（バンドラ不要）
- **アセット**: すべてコード描画（スプライト画像なし）
- **テスト**: Bun のネイティブテストランナー

## 画面仕様

- **基準解像度**: 360×640（9:16 縦長）
- **描画方式**: オフスクリーン Canvas（360×640）→ nearest-neighbor 補間で CSS 拡大
- **レイヤー順**: 背景（3層パララックス）→ 弾 → 敵 → 自機 → エフェクト → HUD
- **FPS**: 60fps 固定（requestAnimationFrame + delta time 制御）

## レンダリングとビジュアル

### ピクセルアート方針
- 全スプライトをコード描画（Canvas API）。画像ファイルは使用しない
- 自機・敵・アイテム: 16×16 〜 32×32 px
- 弾: 半径 3〜8px の円＋グロー効果。色分け:
  - 赤: 通常弾
  - 青: 奇数弾（変則軌道）
  - 紫: レーザー
  - 黄: 高速弾
- 背景: 星＋幾何学模様のタイル描画。ステージごとに色調変化

### エフェクト
- **弾トレイル**: 前フレーム位置を数フレーム保持し半透明描画
- **ヒットエフェクト**: 白フラッシュ＋パーティクル爆散（8〜16個）
- **ボス登場**: 画面外スライドイン＋「WARNING」警告表示
- **ボム**: 画面全体フラッシュ → 全敵弾消滅＋爆発エフェクト
- **被弾時**: 自機点滅（無敵2秒）＋画面シェイク
- **画面シェイク**: Canvas 全体の数ピクセルランダムオフセット

### 背景パララックス
- 3層: 遠景（星、0.2x速）／中景（幾何学模様、0.5x速）／近景（破片、1.0x速）
- ステージ進行で色調が濃く不穏に変化

## ゲームメカニクス

### 自機
| 項目 | 仕様 |
|------|------|
| 移動速度 | 通常 4px/frame、低速 2px/frame（Shift長押し） |
| 当たり判定 | 1px点（自機中心） |
| ショット | Zキー押しっぱなし連射（4発/秒） |
| ボム | Xキー。ストック3。全敵弾消去＋2秒無敵＋ボス大ダメージ |
| 残機 | 初期3。被弾-1、0でゲームオーバー |
| 無敵時間 | 被弾後2秒間、点滅＋当たり判定なし |

### パワーアップ
- 敵撃破時 Pアイテムドロップ。最大5段階:
  1. 前方2way
  2. 前方3way
  3. 前方3way + 左右オプション
  4. 前方5way + 左右オプション
  5. 前方5way + ホーミング弾
- 被弾時に1段階ダウン

### スコア
- グレイズ（自機半径15px以内を敵弾通過）: +100点、連続で倍率上昇
- 敵撃破: 雑魚100〜500、ボス10,000〜50,000
- ボム温存ボーナス: 未使用ボム×5,000（ステージクリア時）
- 残機ボーナス: 残機×10,000（ステージクリア時）

### 敵設計
- **雑魚敵（3タイプ）**: 直進型、自機狙い弾型、画面端往復型
- **中ボス（各面1体）**: 2〜3種の弾幕パターン
- **ボス（各面1体）**: HPでフェーズ移行。最終面ボスは5フェーズ以上

### 難易度進行
- ステージ1〜2: 遅めの弾、単発パターン。チュートリアル的
- ステージ3〜4: 弾数増加、2種同時発動
- ステージ5: 3種同時発動、レーザー＋全方位複合。高難度

## 入力

### キーボード
- 矢印キー: 移動
- Z: ショット
- X: ボム
- Shift: 低速移動（当たり判定可視化）
- Escape: ポーズ

### タッチ（スマートフォン）
- ドラッグ: 自機移動（指に追従）
- ショット: 自動連射（常時）
- ボム: 画面下部のボムボタン（タップ）

## UI 画面一覧

### タイトル画面
- タイトルロゴ「弾幕煉獄」をピクセルフォントで中央表示
- 背景に星パララックス
- 「PRESS Z TO START」点滅
- © 2026 表記

### HUD
- 画面上部に半透明黒バー
- SCORE / HI（ハイスコア）/ 残機（♥）/ ボム（★）/ POWER 表示
- グレイズ時スコア色変化

### ステージ間演出
- クリア時: 「STAGE CLEAR」＋ボーナス加算アニメーション（2秒）
- 開始時: ステージ番号＋ステージ名表示（例: 「STAGE 3 - CRIMSON SKY」）

### ゲームオーバー画面
- 「GAME OVER」赤点滅
- スコア、ハイスコア表示。更新時「NEW RECORD!」
- Z: コンティニュー / X: タイトルへ

### ポーズ画面
- 半透明オーバーレイ＋「PAUSED」表示
- Escape で再開

## 開発方針

1. ゲームエンジンのコア（ゲームループ、Canvas、入力）を最初に構築
2. エンティティ基底クラスと弾システムを次に構築
3. 弾幕パターンを1つずつ実装
4. ステージを順に実装
5. UI画面を実装
6. エフェクト・演出を仕上げ
7. テスト・バランス調整
